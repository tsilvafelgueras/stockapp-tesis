Punto 1 — Etapa 3

🧠 Extracción con IA

OBJETIVO

Eliminar la carga manual de planillas de tintorería. El operario saca una foto o sube el PDF y el sistema extrae los rollos automáticamente.

HERRAMIENTAS

Google Gemini 2.5 Flash @google/genai SDK Next.js Server Actions

FUNCIONALIDADES

  • Foto o PDF de planilla → extracción automática de rollos, metros y kilos
  • Configuración por tintorería — prompt customizado para Muter (diferente al genérico)
  • Revisión y edición antes de confirmar el lote
  • Manejo de error con timeout de 30s y mensaje claro al usuario
Resultado
Precisión >90% en planillas reales de Muter Textil. El operario corrige excepciones, no transcribe manualmente.
Aprendizaje
Un prompt específico por tintorería supera ampliamente al prompt genérico. La IA necesita contexto del formato del proveedor.
📋

Flujo completo

Subir imagen/PDF → Gemini extrae → tabla editable → confirmación → rollos en sistema con QR asignado

Punto 1 — Etapa 4

📱 Scanner QR/Barcode

OBJETIVO

Cerrar la brecha entre el mundo físico y el sistema digital. Cada rollo físico tiene un código QR; el sistema no da por ingresado ningún rollo que no fue escaneado.

HERRAMIENTAS

@zxing/browser Web Camera API Next.js PWA

FUNCIONALIDADES

  • Flujo "Aduana" — confirmación obligatoria rollo por rollo desde el celular
  • Asignación de ubicación física al confirmar (A-1, B-3, hasta F-6)
  • Fallback de ingreso manual si la cámara no está disponible
  • Scanner también en picking — el mismo flujo para despachar
Resultado
0 rollos en estado "fantasma" post-confirmación. Si no está escaneado, no está en stock.
Aprendizaje
Hacer el paso obligatorio (no salteable) cambió el comportamiento del operario. La UX de "no hay atajos" es una feature.
🔄

Estado del rollo

pendiente → en_stock → reservado → entregado

Cada transición requiere acción explícita del usuario con el scanner.

Punto 1 — Etapas 5 y 6

📦 Stock y Pedidos — ciclo end-to-end

VISTA DE STOCK (ETAPA 5)

  • Filtros multicriteria: artículo, color, tintorería, ubicación, estado
  • Vista rollo por rollo con peso, metros, tintorería de origen
  • Accesible para todos los roles (operario, ventas, admin)

MÓDULO DE PEDIDOS (ETAPA 6)

  • Ventas crea el pedido y selecciona rollos disponibles
  • Rollos quedan en estado reservado — no se pueden asignar a otro pedido
  • Operario hace el picking escaneando rollo por rollo
  • Admin confirma la entrega o cancela con devolución de stock
Resultado
Ciclo completo sin papel, sin WhatsApp, sin Excel. Del ingreso a la entrega, todo trazado en el sistema.
Aprendizaje
Las RPCs en Supabase para lógica transaccional del picking son más seguras que lógica en cliente — los estados no se pueden saltar.
👥

4 roles con permisos diferenciados

operario · ventas · admin · super — enforceados a nivel de base de datos con Row Level Security

Punto 1 — Etapa 7

📊 Muestras, Reportes y UX

🧵

Módulo Muestras

Registro de muestras enviadas con descuento automático de kilos del rollo. Trazabilidad: qué se mandó como muestra y cuándo.

📈

Reportes para el Admin

Stock por artículo y color, movimientos del mes, diferencias de inventario, antigüedad de crudo. Export a CSV.

UX Mobile-First

Touch targets de 44px, diseño optimizado para el depósito. Toasts de confirmación (Sonner). Recuperación de contraseña.

Punto 1 — Iteración final

🔧 Iteración de correcciones y features

Tras el primer uso real con Muter Textil: 9 bugs resueltos + 9 features nuevas solicitadas por el cliente.

BUGS CRÍTICOS RESUELTOS

  • Timeout de 30s en IA — antes quedaba en carga infinita con archivos inválidos
  • Permisos: operario ya no puede agregar tintorerías (solo admin)
  • Reportes con etiquetas profesionales en español (antes mostraban rollos.created)
  • Pedidos clickeables desde perfil admin (problema de overlay de z-index)
  • Responsive desktop del depósito corregido

NUEVAS FEATURES (cliente)

  • Ubicaciones de stock — asignación masiva A-1 a F-6 al ingresar un lote
  • Pedidos pendientes — alertas por días (verde <7 / amarillo 7-15 / rojo >15)
  • Estado "segunda" — mercadería defectuosa recuperable, diferente a baja definitiva
  • Cálculo de merma — kg y % por rollo, panel de análisis en reportes
  • Stock mínimo con alertas — badge en dashboard, hook notificarStockMinimo() listo para email/WhatsApp
Resultado
MVP estable y robusto, listo para piloto real con cliente pagante.
Aprendizaje
El primer usuario real revela edge cases que ningún test de laboratorio anticipa. El feedback de Muter fue más valioso que cualquier hipótesis previa.

Punto 2 — El problema

El proceso hoy en una PyME textil

"Yo sé lo que compré. No sé lo que tengo."

— Hernán, dueño de Muter Textil
📦

Stock invisible

Excel desactualizado + WhatsApp con la tintorería. "Cuántos rollos hay" depende de con quién hables y cuándo actualizaron la planilla.

⚠️

Merma sin control

Cuánto se pierde en el proceso de teñido, nadie lo sabe con exactitud. La diferencia entre kilos enviados y kilos recibidos se anota a mano, si se anota.

🔄

Pedidos sin trazabilidad

No hay reserva de stock. El mismo rollo puede quedar prometido a dos clientes. Los pedidos pendientes viven en un anotador o en la memoria.

Costo oculto
Horas-persona de carga manual · errores de despacho · pérdidas por antigüedad de crudo · imposibilidad de responder pedidos urgentes en tiempo real

Punto 2 — Propuesta de valor

Por qué StockApp resuelve algo real

Ningún competidor cubre las cinco piezas críticas del flujo simultáneamente.

🧠

IA entrenada por tintorería

No extracción genérica. Cada proveedor tiene su propio prompt optimizado para su formato de planilla. Flexxus y Tango no tienen esto; lo harían con desarrollo a medida.

📱

Mobile-first para el operario, sin fricción

El operario usa el celular que ya tiene. Sin hardware extra, sin capacitación de semanas. El scanner funciona con la cámara nativa.

🔐

Multi-tenant desde el diseño

Dato de un cliente nunca es visible para otro (Row Level Security por empresa_id). Escala a N clientes sin re-arquitectura.

📍

Ubicaciones físicas reales (A-1 a F-6)

No un campo de texto libre. El sistema conoce el layout del depósito y guía el picking por ubicación.

⚡

Implementación sin consultor · $0 de infraestructura en piloto

Vercel + Supabase free tier. Sin fee de setup obligatorio para empezar. Tango cobra USD 150 – 1.500 solo para instalar.
