Punto 1 — Etapa 3

🧠 Extracción con IA

OBJETIVO

Eliminar la carga manual de planillas de tintorería. El operario saca una foto o sube el PDF y el sistema extrae los rollos automáticamente.

HERRAMIENTAS

Google Gemini 2.5 Flash @google/genai Server Actions

FUNCIONALIDADES

  • Foto o PDF → extracción automática de rollos, metros y kilos
  • Configuración por tintorería — prompt customizado por proveedor
  • Revisión y edición antes de confirmar el lote
  • Timeout de 30s + mensaje claro si la IA no puede interpretar
Resultado
Precisión >90% en planillas reales de Muter Textil. El operario corrige excepciones, no transcribe manualmente.
Aprendizaje
Un prompt específico por tintorería supera ampliamente al genérico. La IA necesita contexto del formato del proveedor.
📋

Flujo completo

Subir imagen/PDF → Gemini extrae → tabla editable → confirmación → rollos en sistema con QR asignado

Punto 1 — Etapa 4

📱 Scanner QR/Barcode

OBJETIVO

Cerrar la brecha entre lo físico y lo digital. Cada rollo tiene QR; el sistema no da por ingresado ningún rollo que no fue escaneado.

HERRAMIENTAS

@zxing/browser Web Camera API PWA mobile

FUNCIONALIDADES

  • Flujo "Aduana" — confirmación obligatoria rollo por rollo
  • Asignación de ubicación física al confirmar (A-1 a F-6)
  • Fallback de ingreso manual si la cámara no está disponible
  • Scanner reutilizado en picking — mismo flujo para despachar
Resultado
0 rollos "fantasma" post-confirmación. Si no se escaneó, no está en stock.
Aprendizaje
Hacer el paso obligatorio (no salteable) cambió el comportamiento del operario. La UX de "sin atajos" es una feature.
🔄

Estado del rollo

pendiente → en_stock → reservado → entregado

Cada transición requiere acción explícita con el scanner.

Punto 1 — Iteración final

🔧 Iteración de correcciones y features

Tras el primer uso real con Muter: 9 bugs resueltos + 9 features nuevas pedidas por el cliente.

BUGS CRÍTICOS RESUELTOS

  • Timeout de 30s en IA — antes carga infinita con archivos inválidos
  • Permisos: operario ya no puede agregar tintorerías
  • Reportes con etiquetas en español (antes rollos.created)
  • Pedidos clickeables desde perfil admin (overlay de z-index)
  • Responsive desktop del depósito corregido

NUEVAS FEATURES (cliente)

  • Ubicaciones de stock — asignación masiva A-1 a F-6
  • Pedidos pendientes — alertas por días (verde/amarillo/rojo)
  • Estado "segunda" — defectuosa recuperable, distinta a baja
  • Cálculo de merma — kg y % por rollo, panel en reportes
  • Stock mínimo con alertas — hook listo para email/WhatsApp
Resultado
MVP estable y robusto, listo para piloto real con cliente pagante.
Aprendizaje
El primer usuario real revela edge cases que ningún test de laboratorio anticipa. El feedback de Muter fue más valioso que cualquier hipótesis previa.

Punto 2 — El problema

El proceso hoy en una PyME textil

"Yo sé lo que compré. No sé lo que tengo."

— Hernán, dueño de PyME textil entrevistada en validación
📦

Stock invisible

Excel desactualizado + WhatsApp con la tintorería. "Cuántos rollos hay" depende de con quién hables y cuándo se actualizó la planilla.

⚠️

Merma sin control

Cuánto se pierde en el teñido nadie lo sabe con exactitud. La diferencia entre kilos enviados y recibidos se anota a mano, si se anota.

🔄

Pedidos sin trazabilidad

No hay reserva de stock. El mismo rollo puede quedar prometido a dos clientes. Los pedidos pendientes viven en un anotador o en la memoria.

Costo oculto
Horas-persona de carga manual · errores de despacho · pérdidas por antigüedad de crudo · imposibilidad de responder pedidos urgentes

Punto 3 — Cliente ideal

Foco angosto: el cliente ideal

Tres condiciones que tienen que darse juntas. Capturar bien este segmento antes de ampliar.

🏭

1. Terceriza el teñido

Envía tela cruda a tintorería externa y la recibe terminada. Es el flujo donde se pierde la trazabilidad y donde StockApp aporta valor.

📏

2. Tamaño mediano

De 5 a 50 empleados. Volumen de stock que justifica el sistema. Más chicas se arreglan con una planilla; más grandes ya tienen un ERP.

📋

3. No tiene sistema

Gestiona con Excel, libreta o memoria. No hay que desplazar a un competidor: reemplazar una solución manual que no funciona.

"El producto es para gente que no produce sus telas, que terceriza todo."

— Dalmiro, dueño de Muter Textil (cliente piloto)

Fuera del foco: empresas grandes con ERP propio · empresas con tintorería propia · confeccionistas puras (no manejan rollos de tela).
